# Design System Evaluation

Meta-knowledge for building, evaluating, and governing design systems. Synthesized from awesome-design-systems (200+ real-world systems).

## When to Use

- Evaluating an existing design system's maturity
- Creating a new design system from scratch
- Auditing against industry gold standards
- Establishing governance and contribution processes

## 4-Pillar Maturity Model

| Pillar | What It Covers | Mature? |
|--------|---------------|---------|
| Components | Coded UI patterns with interactive examples | Required |
| Voice & Tone | Communication/brand language guidance | Differentiator |
| Design Kits | Figma/Sketch files for designer collaboration | Expected |
| Source Code | Public repos enabling transparency | Best practice |

## Gold Standards

| System | Strength | Best For |
|--------|----------|----------|
| IBM Carbon | Governance + accessibility | Enterprise |
| Shopify Polaris | Contribution guides | Commerce |
| Google Material | Multi-platform | Cross-platform |
| Adobe Spectrum | Enterprise language | Complex tools |
| Microsoft Fluent | Cross-platform | Windows + web |
| GitHub Primer | Developer tools | DevX |

## Quick Maturity Score

Rate each pillar 0-3. Sum = maturity (0-12). 0-3 Nascent, 4-6 Developing, 7-9 Mature, 10-12 Exemplary.
